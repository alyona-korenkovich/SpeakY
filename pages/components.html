Компоненты

Компонент – это элемент или блок, который можно редактировать комплексно. Задав его вид один раз, автоматически измененные копии компонента можно использовать в разных частях проекта.

Например, если вы нарисуете иконку и сделаете из нее компонент, а потом скопируете ее, то при изменении цвета, размера и других параметров главного компонента, копия будет изменяться соответственно.

Аватар

Аватар используется для отображения личности одного пользователя или сущности. Иллюстрация аватара по умолчанию отображается как на рисунке ниже.

Баннер

Баннер информирует пользователей о важных изменениях или условиях в интерфейсе. Используйте этот компонент, если вам нужно общаться с пользователями в заметной форме.

Вкладка

Интерактивная кнопка вкладок для использования в компоненте группы вкладок.

Выпадающий список

В раскрывающемся меню отображается список действий или доступных для выбора параметров для пользователя.

Календарь

Календарь позволяет пользователю выбрать дату. Увидеть дедлайны и текущую дату.

Карточка

Карточка — это затененная поверхность, на которой отображается содержимое и действия по одной теме. Её должно быть легко сканировать на предмет актуальной и полезной информации.

Кнопка

Кнопки – это один из наиболее известных интерактивных элементов пользовательского интерфейса.

Это интерактивный объект, который позволяет получить желаемый отклик от сервиса/приложения после выполнения определенной команды. По сути, кнопка является элементом управления, дающим возможность пользователю напрямую общаться с устройством и отправлять необходимые команды для достижения тех или иных целей, например: отправить электронное письмо, купить какой-то товар.

Лоадер

Компонент лоадер используется для того, чтобы указать пользователям, что их действие обрабатывается.

Модальный компонент

Модальный компонент используется для отображения контента, который временно блокирует взаимодействие с основным представлением приложения. Модальные средства следует использовать умеренно и только в случае необходимости.

Навигация

Переключатель навигации предназначен для скрытия и отображения основной навигации. Этот компонент используется внутри компонента Layout, но также может использоваться отдельно для дальнейшей настройки поведения.

Навигационная группа

Навигационная группа включает в себя все действия или элементы в одной группе и используется для группировки элементов в связанные категории.

Обмен сообщениями

Сообщение представляет собой определенный элемент коммуникации системы и пользователя, например уведомления, задачи или беседы. Отправка обратных сообщений осуществляется с помощью выбора ответов кнопками.

Поле ввода

Входные данные используются для того, чтобы пользователи могли вводить текст, данные или сообщения.

Прогресс

Индикатор выполнения используется для визуального представления завершения задачи или процесса. Он показывает, какая часть задачи выполнена, а какая еще осталась.

Радио-кнопка

Переключатели — это графические элементы пользовательского интерфейса, которые позволяют пользователю выбрать только один вариант из предопределенного набора взаимоисключающих опций.

Пример

Разделитель

Компоненты-разделители используются для разделения и различения разделов контента или групп пунктов меню. Визуально они выглядят как горизонтальные или вертикальные линии.

Скелет

Скелеты используются для обеспечения представления содержимого с низкой точностью до того, как оно появится в представлении. Это улучшает воспринимаемое время загрузки для наших пользователей.

Сообщение

Сообщение представляет собой определенный элемент в коллекции, например уведомления, задачи или беседы. Сообщение может быть размещено непосредственно внутри выпадающего компонента.

Тумблер

Тумблер позволяет управлять функцией или параметром, который можно включать или выключать. Если физический переключатель будет работать для действия, переключатель, вероятно, будет лучшим компонентом для использования.

Уведомление

Уведомление содержит важную информацию, требующую действий или подтверждения. Уведомление отображается до тех пор, пока пользователь не закроет его.

Уведомление об ошибке

Пустое состояние можно использовать, когда нет данных для отображения, чтобы описать, что пользователь может сделать дальше. Пустое состояние предоставляет пояснения и рекомендации, помогающие пользователю двигаться вперед.

Хештег

Хештег это ключевое слово, тип пометки или тега, облегчающий поиск информации по теме или содержанию.

Чек-бокс

Флажки позволяют пользователю выбрать один или несколько вариантов из ограниченного набора вариантов.
